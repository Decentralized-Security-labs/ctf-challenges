# DSEC Labs CTF

A growing series of hands-on security challenges from
[DSEC Labs](https://dseclabs.xyz). Each one is a small, self-contained target you
run on your own machine and try to break. They exist to help you learn and train
real offensive skills across **AI**, **Web3**, and **Web2**: from prompt injection
against LLM apps, to smart-contract exploitation, to classic application security.

New challenges land here over time, and each one gets a full writeup on our
[blog](https://dseclabs.xyz/blog) once it has had time in the wild, so you can
measure your solution against the intended path.

## How to play

Every challenge lives in its own folder with a README that carries the story, the
objective, and a one-line command to start it. The challenges ship as prebuilt
Docker images, so all you need is Docker: pull, run, and start poking.

Solve each one through its intended interface. Pulling the flag out of the
container or the image (reading files, dumping layers, and so on) is out of scope;
the point is the technique, not the shortcut. Each challenge README spells out its
own rules of engagement.

## Who this is for

Anyone who wants to practice offensive security in a safe, legal sandbox: people
studying for AI red teaming, Web3 auditing, or web and application pentesting, and
practitioners who just want a target to sharpen a specific skill against. Start at
Tier 0 and work up.
